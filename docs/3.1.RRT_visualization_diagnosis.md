# RRT可视化问题诊断报告

## 问题1：看不到RRT树的蓝色线条

### 根本原因
RViz没有订阅 `/shapes` 话题来显示RRT树的可视化。

### 证据
```bash
ros2 topic info /shapes --verbose
# 输出显示：
# Publisher count: 2  ✅ (global_rrt_detector + local_rrt_detector)
# Subscription count: 0  ❌ (RViz没有订阅)
```

### ROS1原版配置
在 `yahboomcar_nav_rrt_ros1/rviz/rrt_map.rviz` 中有：
```yaml
- Class: rviz/Marker
  Marker Topic: /global_detector_shapes
  Name: Marker

- Class: rviz/Marker
  Marker Topic: /local_detector_shapes
  Name: Marker
```

### 解决方案
**在RViz中手动添加Marker显示：**

1. 打开RViz
2. 点击左下角 "Add" 按钮
3. 选择 "By topic" 标签
4. 找到 `/shapes` → `Marker`
5. 点击 "OK"

或者修改当前使用的RViz配置文件，添加：
```yaml
- Class: rviz/Marker
  Enabled: true
  Marker Topic: /shapes
  Name: RRT_Tree
  Namespaces:
    markers: true
  Queue Size: 100
```

## 问题2：点击后没有标记点显示

### 根本原因
当前代码在收集完5个点后会清空点的显示。

### 代码位置
`global_rrt_detector_ros2.cpp` 第85-86行：
```cpp
points_.points.clear();  // 清空了点的显示
shapes_pub_->publish(points_);
```

### ROS1原版行为
ROS1版本也是清空的，但教程中的标记点可能来自：
1. **Publish Point工具本身的可视化**
2. **另一个独立的Marker话题**（如 `/clicked_point_markers`）

### 当前话题状态
```bash
/clicked_point  # 有数据，但这是PointStamped类型，不是可视化Marker
/shapes         # 只显示RRT树线条，点被清空了
```

---

## 建议给Codex的修复方案

### 方案1：保留点击点的可视化（推荐）
修改 `global_rrt_detector_ros2.cpp`，不要清空前5个点：
```cpp
// 第85-86行，改为：
// 只清空临时可视化，保留前5个定义点
visualization_msgs::msg::Marker region_points = points_;
region_points.id = 2;  // 使用不同ID
region_points.color.g = 1.0;  // 改为绿色
shapes_pub_->publish(region_points);  // 发布区域点

points_.points.clear();  // 然后清空用于RRT树
```

### 方案2：RViz配置修复（立即可用）
在RViz中添加 `/shapes` 的Marker显示即可看到RRT树。

**点击标记的问题：**
