# yahboomcar_nav_rrt_ros2

ROS2 自主探索导航包，基于 RRT（Rapidly-exploring Random Tree）算法实现未知环境的自动探索与建图。

## 功能特性

- **SLAM 建图**：集成 SLAM Toolbox 实现实时建图
- **自主导航**：基于 Nav2 的路径规划与避障
- **多点导航**：支持设置多个目标点顺序导航
- **地图保存**：提供地图保存服务
- **激光雷达支持**：兼容多种激光雷达型号（A1/S2）

## 使用方法

### 完整启动流程（推荐）

**使用一体化launch文件启动**

```bash
# 终端1：启动RRT探索导航（底盘+雷达+SLAM+Nav2）
cd /home/jetson/wbt_ws/WBT_ROS2_WS
source install/setup.bash
ros2 launch yahboomcar_nav_rrt rrt_full.launch.py

# 终端2：启动RViz可视化
cd ~/yahboomcar_ros2_ws/yahboomcar_ws
source install/setup.bash
ros2 launch yahboomcar_nav display_map_launch.py
```

**说明**：
- `rrt_full.launch.py` 会自动启动yahboomcar的`laser_bringup_launch.py`（底盘+雷达）
- yahboomcar已经处理好命名空间转换，不需要手动启动中继节点
- 参考gmapping的成功经验，使用官方launch文件更可靠

### 分步启动方式（不推荐，仅供调试）

```bash
# 设置激光雷达类型（可选：a1 或 s2）
export RPLIDAR_TYPE=a1

# 启动探索节点
ros2 launch yahboomcar_nav_rrt rrt_exploration.launch.py
```

### 2. 多点导航

启动多点导航节点后，在 RViz 中：
1. 使用 "Publish Point" 工具点击地图设置目标点
2. 机器人将按顺序访问所有目标点
3. 发布 `/initialpose` 可清除所有目标点

```bash
ros2 run yahboomcar_nav_rrt send_mark.py
```

### 3. 保存地图

```bash
# 方法 1：调用服务
ros2 service call /save_map std_srvs/srv/Trigger

# 方法 2：直接保存
ros2 run nav2_map_server map_saver_cli -f ~/my_map
```

## 核心节点

### save_map
- **类型**：C++ 节点
- **功能**：提供 `/save_map` 服务保存当前地图
- **服务类型**：`std_srvs/srv/Trigger`

### multipoint_navigation (send_mark.py)
- **类型**：Python 节点
- **功能**：多点导航控制
- **订阅话题**：
  - `/clicked_point` (PointStamped)：接收目标点
  - `/initialpose` (PoseWithCovarianceStamped)：清除目标点
  - `/JoyState` (Bool)：手柄控制状态
- **发布话题**：
  - `/path_point` (MarkerArray)：可视化路径点
  - `/goal_pose` (PoseStamped)：导航目标
- **Action 客户端**：`navigate_to_pose` (Nav2)

### scan_dilute.py
- **类型**：Python 节点
- **功能**：稀释 S2 激光雷达扫描数据以提高性能
- **条件**：仅在 `RPLIDAR_TYPE=s2` 时启动

### topic_relay.py
- **类型**：Python 节点
- **功能**：转换话题命名空间（`/robot1/*` → `/*`）
- **转换话题**：
  - `/robot1/scan` → `/scan`
  - `/robot1/odom` → `/odom`
- **重要**：同时移除消息中的 `robot1/` frame_id 前缀

### tf_relay.py
- **类型**：Python 节点
- **功能**：转换TF坐标系命名空间
- **转换坐标系**：
  - `robot1/odom` → `odom`
  - `robot1/base_footprint` → `base_footprint`
  - `robot1/base_link` → `base_link`
  - `robot1/laser_link` → `laser_link`
- **作用**：解决机器人底盘使用命名空间而Nav2不支持的问题

## 参数配置

Nav2 参数文件位于：`param/common/nav2_params.yaml`

**重要**：该文件是ROS2 Nav2必需的配置文件，包含以下关键组件：
- `bt_navigator`：行为树导航器配置
- `controller_server`：DWB局部规划器配置
- `planner_server`：NavFn全局规划器配置
- `local_costmap`：局部代价地图配置
- `global_costmap`：全局代价地图配置

关键参数：
- `base_frame`: `base_footprint`
- `odom_frame`: `odom`
- `map_frame`: `map`
- `scan_topic`: `scan` 或 `scan_dilute`
- `robot_radius`: `0.22` (机器人半径，米)
- `max_vel_x`: `0.26` (最大线速度，米/秒)

## 环境变量

- `RPLIDAR_TYPE`：激光雷达型号（`a1` 或 `s2`，默认 `a1`）
- `ROBOT_TYPE`：机器人型号（默认 `X3`）

## 实现说明

### ROS1 到 ROS2 转换要点

1. **Nav2 参数文件创建**
   - 原ROS1包使用 `move_base` 和独立的costmap参数文件
   - ROS2需要统一的 `nav2_params.yaml` 配置文件
   - 包含所有Nav2组件的参数（bt_navigator, controller_server, planner_server, costmaps）

2. **Launch文件转换**
   - 从ROS1 XML格式转换为ROS2 Python格式
   - 使用 `IncludeLaunchDescription` 组织子launch文件
   - 通过环境变量 `RPLIDAR_TYPE` 动态选择激光雷达配置

3. **Python节点适配**
   - 从 `rospy` 迁移到 `rclpy`
   - Action客户端API变更（`actionlib` → `rclpy.action`）
   - 话题订阅/发布API更新

## 故障排除

### 缺少 nav2_params.yaml
**症状**：启动时警告 "Parameter file path is not a file"
**解决**：确保 `param/common/nav2_params.yaml` 文件存在并包含所有Nav2组件配置

### 缺少 nav2_params.yaml
**症状**：启动时警告 "Parameter file path is not a file"
**解决**：确保 `param/common/nav2_params.yaml` 文件存在并包含所有Nav2组件配置

### RViz中看不到激光雷达点云
**症状**：RViz启动但没有激光扫描显示
**原因**：TF命名空间不匹配（`robot1/laser` vs `laser`）
**解决**：确保 `topic_relay.py` 和 `tf_relay.py` 正常运行
```bash
# 检查中继节点
ros2 node list | grep relay

# 检查frame_id是否正确
ros2 topic echo /scan --once | grep frame_id
# 应该显示 frame_id: laser（而不是 robot1/laser）
```

### 激光雷达数据过密
如使用 S2 雷达，设置 `RPLIDAR_TYPE=s2` 启用扫描稀释节点。

### 导航失败
检查：
1. TF 树是否完整（`ros2 run tf2_tools view_frames`）
2. 激光雷达数据是否正常（`ros2 topic echo /scan`）
3. Nav2 参数配置是否匹配机器人

### 地图保存失败
确保有写入权限，或手动指定路径：
```bash
ros2 run nav2_map_server map_saver_cli -f /path/to/map
```
