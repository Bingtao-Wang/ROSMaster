## 一、导航原理简述

```
已有地图 + 激光雷达实时扫描
        ↓
   AMCL 定位 (粒子滤波, 将机器人定位到地图中)
        ↓
   Global Planner (全局路径规划, A*/Dijkstra)
        ↓
   TEB Local Planner (局部路径规划, 动态避障)
        ↓
   /cmd_vel → 底盘驱动
```

TEB (Timed Elastic Band) 适合 X3 麦克纳姆轮的原因:
- 支持全向运动模型
- 动态障碍物避让能力强
- 路径平滑, 转弯自然

## 四、RViz 导航操作

### 步骤 1: 设置初始位姿

1. 点击 RViz 工具栏的 **2D Pose Estimate** 按钮
2. 在地图上 **点击并拖动箭头** 指向机器人实际朝向
3. 观察激光扫描 (红色点) 是否与地图墙壁对齐
4. 如果没对齐, 重新设置, 直到激光点与地图吻合

> 初始位姿越准确, AMCL 收敛越快, 导航越可靠

### 步骤 2: 发送导航目标

1. 点击 RViz 工具栏的 **2D Goal Pose** 按钮 (或 Nav2 Goal)
2. 在地图上 **点击并拖动箭头** 指定目标位置和朝向
3. 机器人将自动:
   - 规划全局路径 (绿色线)
   - 沿路径行驶 (蓝色局部路径)
   - 遇到障碍物自动绕行
   - 到达目标后停止

### 步骤 3: 取消导航 (如需要)

```bash
# 命令行取消
ros2 topic pub --once /goal_pose geometry_msgs/PoseStamped  # Ctrl+C 中断

# 或使用手柄控制覆盖 (按任意方向键)
```

## 七、导航参数调优

参数文件位置: `yahboomcar_nav/params/teb_nav_params.yaml`

### 常调参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_vel_x` | 0.5 | 最大前进速度 (m/s), 室内建议 0.3 |
| `max_vel_theta` | 1.0 | 最大旋转速度 (rad/s) |
| `xy_goal_tolerance` | 0.1 | 到达目标的位置容差 (m) |
| `yaw_goal_tolerance` | 0.1 | 到达目标的角度容差 (rad) |
| `inflation_radius` | 0.55 | 障碍物膨胀半径 (m), 越大越保守 |
| `robot_radius` | 0.15 | 机器人半径 (m) |

### AMCL 定位参数

| 参数 | 说明 |
|------|------|
| `min_particles` | 最小粒子数, 越多定位越准但越慢 |
| `max_particles` | 最大粒子数 |
| `update_min_d` | 移动多少米后更新定位 |
| `update_min_a` | 旋转多少弧度后更新定位 |

## 九、常见问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 导航栈启动失败 | 地图文件路径错误 | 确认 `map:=` 指向正确的 `.yaml` 文件 |
| 激光与地图不对齐 | 初始位姿设置不准 | 重新用 2D Pose Estimate 设置 |
| AMCL 粒子发散 | 初始位姿偏差太大 | 手动将机器人开到已知位置, 重设初始位姿 |
| 设完 Goal 机器人不动 | 未设初始位姿, AMCL 无法发布 map→odom TF | **必须先设 2D Pose Estimate**, AMCL 才能工作 |
| bt_navigator 激活失败 | 行为树 XML 引用了未注册的插件 (如 Spin) | 在 `teb_nav_params.yaml` 的 `behavior_plugins` 中添加 `spin` 和 `wait` 插件 (见下方修复方法) |
| Nav2 节点状态混乱 | 多次启动导致重复节点 | 必须先 `killall` 清理所有残留进程再重启 |
| 机器人原地旋转不前进 | 被代价地图判定为困住 | 减小 `inflation_radius` 或清空周围障碍 |
| 路径绕远路 | 全局代价地图有残留障碍 | 清除代价地图: `ros2 service call /global_costmap/clear_entirely_global_costmap nav2_msgs/srv/ClearEntireCostmap` |
| 到不了目标点 | 目标点在障碍物上或地图外 | 选择空白可达区域作为目标 |
| 机器人抖动/振荡 | TEB 参数过于激进 | 降低 `max_vel_x` 和 `max_vel_theta` |
| 导航中途停止 | 恢复行为耗尽 | 查看日志, 手动移开障碍或重发目标 |

### bt_navigator Spin 插件缺失修复方法

默认的 `teb_nav_params.yaml` 中 `behavior_plugins` 只有 `["backup"]`, 但 Nav2 行为树 XML 使用了 `Spin` 和 `Wait` 节点, 导致 bt_navigator 激活失败。

修改 `yahboomcar_nav/params/teb_nav_params.yaml` 中的 behavior_server 部分:

```yaml
# 原始 (会导致 bt_navigator 报 "Node not recognized: Spin"):
behavior_plugins: ["backup"]

# 修复后:
behavior_plugins: ["spin", "backup", "wait"]
spin:
  plugin: "nav2_behaviors/Spin"
backup:
  plugin: "nav2_behaviors/BackUp"
wait:
  plugin: "nav2_behaviors/Wait"
```

修改后需重新编译并完全重启:
```bash
cd ~/yahboomcar_ros2_ws/yahboomcar_ws
colcon build --packages-select yahboomcar_nav --symlink-install
# 杀掉所有 ROS2 进程后重新按顺序启动
```
