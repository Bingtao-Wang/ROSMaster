# GMapping SLAM 建图操作文档

> 适用平台: X3 (麦克纳姆轮) + A1/S2 激光雷达
> 框架: ROS2 Humble
> 工作空间: `~/yahboomcar_ros2_ws/yahboomcar_ws`

---

## 一、前置准备

### A1 雷达
```bash
cd ~/yahboomcar_ros2_ws/yahboomcar_ws
source install/setup.bash
export ROBOT_TYPE=x3
export RPLIDAR_TYPE=a1
```

### S2 雷达
```bash
cd ~/yahboomcar_ros2_ws/yahboomcar_ws
source install/setup.bash
export ROBOT_TYPE=x3
export RPLIDAR_TYPE=s2
```

> 每个新终端都需要执行 `source install/setup.bash`

## 二、启动步骤

### A1 雷达启动方式

#### 终端 1: 启动 GMapping (底盘 + A1雷达 + SLAM)
```bash
ros2 launch yahboomcar_nav map_gmapping_launch.py rplidar_type:=a1
```

该命令自动启动:
- X3 底盘驱动 (Mcnamu_driver_X3)
- 里程计节点 (base_node_X3)
- IMU 滤波 (imu_filter_madgwick) + EKF 融合
- A1 雷达驱动 (sllidar_launch)
- GMapping SLAM 节点
- 手柄控制 (yahboom_joy_X3 + joy_node)

### S2 雷达启动方式

#### 终端 1: 启动 GMapping (底盘 + S2雷达 + SLAM)
```bash
ros2 launch yahboomcar_nav map_gmapping_launch.py rplidar_type:=s2
```

该命令自动启动:
- X3 底盘驱动 (Mcnamu_driver_X3)
- 里程计节点 (base_node_X3)
- IMU 滤波 (imu_filter_madgwick) + EKF 融合
- S2 雷达驱动 (sllidar_s2_launch)
- 扫描降采样 (scan_filter, S2 点数 >1440 需过滤)
- GMapping SLAM 节点 (订阅 /downsampled_scan)
- 手柄控制 (yahboom_joy_X3 + joy_node)

### 终端 2: 启动 RViz 可视化

```bash
ros2 launch yahboomcar_nav display_map_launch.py
```

### 终端 3: 键盘遥控 (如果没有手柄)

```bash
ros2 run yahboomcar_ctrl yahboom_keyboard
```

## 三、键盘操作说明

### 移动控制 (九宫格)

```
u — 左前方    i — 前进    o — 右前方
j — 左转      k — 停止    l — 右转
m — 左后方    , — 后退    . — 右后方
```

### 速度调节

| 按键 | 功能 |
|------|------|
| `q` / `z` | 线速度 + 角速度 同时 增加/减少 10% |
| `w` / `x` | 只 增加/减少 线速度 10% |
| `e` / `c` | 只 增加/减少 角速度 10% |

### 特殊功能

| 按键 | 功能 |
|------|------|
| `t` / `T` | 切换 X/Y 方向速度 (麦克纳姆轮横移) |
| `s` / `S` | 退出键盘控制 |
| `空格` / `k` | 急停 |
| 其他任意键 | 平滑减速停止 |

## 四、建图技巧

1. **速度要慢** — 建议先按 `z` 几次降低速度, 慢速移动建图质量更好
2. **多转几圈** — 确保环境被充分扫描, 特别是拐角处
3. **回环闭合** — 走一圈回到起点, GMapping 会自动修正累积误差
4. **避免快速旋转** — 角速度过快会导致扫描匹配失败
5. **观察 RViz** — 如果地图出现错位/重影, 说明速度太快或里程计漂移

## 五、保存地图

建图满意后, 新开终端执行:

```bash
cd ~/yahboomcar_ros2_ws/yahboomcar_ws && source install/setup.bash
mkdir -p ~/yahboomcar_ros2_ws/yahboomcar_ws/maps
```

### 推荐方式 (已验证可用)

GMapping 的 `/map` 话题 QoS 为 `VOLATILE`, 使用 launch 文件保存会出现 QoS 不兼容报错。
需使用以下命令并指定 `map_subscribe_transient_local:=false`:

```bash
ros2 run nav2_map_server map_saver_cli \
  -f /home/jetson/yahboomcar_ros2_ws/yahboomcar_ws/maps/my_map \
  --ros-args -p map_subscribe_transient_local:=false -p save_map_timeout:=10000.0
```

### 已知问题: launch 方式 QoS 不兼容

以下命令会因 DURABILITY_QOS_POLICY 不匹配而失败, **不要使用**:
```bash
# ✗ 会报 QoS 不兼容错误
ros2 launch yahboomcar_nav save_map_launch.py map_path:=xxx
```

### 保存结果

生成文件:
- `maps/my_map.pgm` — 栅格地图图片 (384x384, 分辨率 0.05m/像素)
- `maps/my_map.yaml` — 地图元数据

`my_map.yaml` 内容示例:
```yaml
image: my_map.pgm
mode: trinary
resolution: 0.05        # 米/像素
origin: [-10, -10, 0]   # 地图左下角坐标 [x, y, yaw]
negate: 0
occupied_thresh: 0.65   # 占用概率 > 0.65 视为障碍物
free_thresh: 0.25       # 占用概率 < 0.25 视为自由空间
```

## 七、常见问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| RViz 无地图显示 | Fixed Frame 不对 | RViz 左侧 Fixed Frame 设为 `map` |
| 地图不更新 | 机器人未移动或 scan 无数据 | 检查 `ros2 topic hz /scan` |
| 雷达无数据 | USB 未连接或权限问题 | `ls /dev/ttyUSB*` 确认设备, `sudo chmod 666 /dev/ttyUSB0` |
| 底盘不动 | 驱动板未上电或串口错误 | 检查电源开关和 USB 连接 |
| 地图有重影 | 移动太快 | 降低速度, 按 `z` 减速 |
| 保存地图失败 | map_server 未安装 | `sudo apt install ros-humble-nav2-map-server` |
| 保存地图 QoS 报错 | GMapping `/map` 是 VOLATILE, launch 默认 TRANSIENT_LOCAL | 用 `map_saver_cli` 加 `-p map_subscribe_transient_local:=false` |
| 保存地图 "Failed to spin" | `/map` 话题无数据或超时 | 确认 GMapping 在运行, 加 `-p save_map_timeout:=10000.0` |
